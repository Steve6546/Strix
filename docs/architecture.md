# التصميم المعماري المقترح

## القرار التقني

أفضل اختيار عملي لهذا النظام:

- Discord bot: TypeScript + discord.js.
- Database: PostgreSQL.
- ORM/Migrations: Prisma.
- Dashboard: Express كبداية، ويمكن نقله لاحقاً إلى Next.js إذا احتجت واجهة أكبر.
- Background workers: تبدأ داخل نفس العملية، ثم تفصل لاحقاً إلى Worker مستقل عند التوسع.

## مبادئ التنفيذ

- كل إعدادات السيرفر تحفظ في قاعدة البيانات، ولا توجد قيم ثابتة تخص سيرفر معين داخل الكود.
- كل حدث Discord يعالج بمعرّف Idempotency لمنع الاحتساب المكرر.
- تحديث الستريك يتم داخل Transaction واحدة.
- Discord API calls تكون قليلة قدر الإمكان، مع Cache للإعدادات والرتب والقنوات.
- Dashboard لا يسمح بأي تعديل إلا لمستخدم لديه صلاحية Administrator أو صلاحية مخصصة.

## تدفق احتساب رسالة

1. استقبال `messageCreate`.
2. تجاهل البوتات والرسائل غير المؤهلة حسب إعدادات السيرفر.
3. إنشاء `ActivityEvent` بمعرّف فريد.
4. محاولة إدخاله في قاعدة البيانات.
5. إذا كان مكرراً، يتم تجاهله.
6. حساب وزن النشاط.
7. تحديث تقدم اليوم للعضو.
8. إذا وصل الحد اليومي، يتم إكمال اليوم وزيادة الستريك عند الحاجة.
9. تسجيل العملية في Audit Log.

## Calendar Reset مقابل Rolling 24 Hours

أنصح أن يكون الافتراضي `Calendar Reset` لأنه أوضح للإدارة والأعضاء وأسهل في الدعم. `Rolling 24 Hours` مفيد للسيرفرات الصغيرة، لكنه يسبب أسئلة كثيرة من المستخدمين لأن وقت الانقطاع يختلف لكل عضو.

## ProBot Credits

هذه الميزة يجب أن تكون مرحلة لاحقة. سبب ذلك أن التحقق الآلي من تحويلات ProBot يعتمد على طريقة ظهور رسائل ProBot وسياسات Discord، وهذا تكامل قابل للكسر. الأفضل تصميمه كـ Payment Provider منفصل:

- `restore_requests`
- `external_payment_events`
- `provider = probot`
- تحقق Idempotent باستخدام معرف عملية أو بصمة رسالة التحويل.

## النسخ الاحتياطي

في الإنتاج لا تعتمد على Backup من داخل البوت فقط. الأفضل:

- نسخ PostgreSQL خارجية عبر `pg_dump`.
- تشفير النسخ.
- اختبار استعادة دوري.
- احتفاظ بنسخ يومية وأسبوعية.

## واجهة Discord

داخل Discord استخدم:

- Slash commands للأوامر الأساسية.
- Buttons للعمليات السريعة.
- Select menus للإعدادات ذات الخيارات.
- Modals للقيم النصية والرقمية.
- Embeds مرتبة للعرض.

لوحة الويب تكون للإعدادات الثقيلة والجداول والبحث في السجلات.
